# Architecture Decision Records

## ADR-001

**Decision:** Use FastAPI as the backend framework.

**Reason:** Fast development, automatic OpenAPI documentation, asynchronous support, and strong Python ecosystem.

---

## ADR-002

**Decision:** Use Mistral API for AI capabilities.

**Reason:** Fast inference, lower cost, strong instruction following, and simple REST integration.

---

## ADR-003

**Decision:** Use modular monolith instead of microservices.

**Reason:** Hackathon timeline, reduced deployment complexity, and easier debugging.

---

## ADR-004

**Decision:** Store cybersecurity knowledge in JSON.

**Reason:** Human-readable, version-controlled, easy to edit, and sufficient for MVP.

---

## ADR-005

**Decision:** Backend generates recommendations. AI explains recommendations.

**Reason:** Deterministic business logic, explainable results, reduced hallucinations, and easier testing.
